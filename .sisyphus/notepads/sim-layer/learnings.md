## Consumer Agent Implementation (Task 9)
- Purchase funnel is implemented as a multi-stage process: browse -> view -> cart -> purchase.
- Stage transitions are probabilistic based on consumer profile (interest, price sensitivity, urgency).
- Browse step emits Mode 1 correlated reads (`browse_catalog`, `check_reviews`).
- View step emits Mode 2 Ch.3 query (`product_details`).
- Purchase intent emits Ch.1 event and Ch.2 message (`place_order`).
- OrderAccepted triggers Payment message.
- ShipmentNotification marks order as delivered.
- Cancel pipeline uses `order_history` query to identify late orders and sends `cancel_request`.

### Government Agent Implementation (Task 10)
- Government Agent uses a two-tick pipeline for statistics: QueryRequest (tick N) and QueryResult processing (tick N+1).
- It handles both order and disruption reports by inserting them into `gov_records`.
